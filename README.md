Mini-projeto voltado para a matéria de Testes, aqui nele aplicaremos os princípios que aprenderemos. 
Primeira Parte:
Cada integrante do grupo deverá construir no mínimo 20 testes unitários. Lembrem-se de realizar suite testes/ integração.

Requisitos para as equipes que não utilizarem o TCC:<br>

Desenvolver um sistema<br>
1 - Tela de Login<br>
2 - Usuário Admin<br>
3 - Cadastro de usuário (Nome completo, ID, E-mail, Telefone)<br>
4 - Remoção de usuário<br>
5 - Cadastro de produtos (3 tipos)<br>
6 - Remoção de produtos<br>
7 - Compra e Venda<br>
8 - Atualização de estoque<br>
9 - Teste Unitário/Integração<br>

### Lembrem-se de construir um sistema capaz de evoluir para uma interface gráfica.<br>
Segunda Parte Requer:<br>
• Criar no mínimo 30 casos de teste (Por aluno)<br>
• Incluir testes:<br>
o Funcionais (ex: login, cadastro, exclusão)<br>
o De validação (ex: campos obrigatórios, formatos inválidos)<br>
• Utilizar linguagem clara e objetiva<br>
• Garantir que os testes possam ser reproduzidos por outra pessoa<br>

Entrega<br>
• Entregar a planilha preenchida<br>
• Relatório curto contendo:<br>
o Principais erros encontrados<br>
o Dificuldades enfrentadas<br>
o Sugestões de melhoria no sistema<br>
